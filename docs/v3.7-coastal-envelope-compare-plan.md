# v3.7 Coastal Envelope Compare Plan

This note records the agreed next geometry experiment after the failed water-edge runtime-masking approach.

The goal is to compare two preprocessing-driven product families that both preserve true internal ICB/HB boundaries while replacing the outer coast with a cleaner official external-envelope definition.

## What We Are Trying To Achieve

The target visual contract is:

- internal ICB/HB boundaries keep their true canonical geometry
- outer coastal borders follow a cleaner official clipped/generalised coastline character
- inland projecting waterways are removed from the visible outer coast
- the land/sea mask then follows that same rebuilt outer coast
- facilities remain inside the correct shipped board products after regeneration

The failed runtime water-edge modifier path showed that post-hoc line dimming cannot reliably redefine the true border endpoints.

So the next step is preprocessing, not more runtime masking.

## Two Compare Products

### 1. `bfe` external-envelope product

Use canonical board products for internal borders, but replace the outer coast against the more seaward official `BFE` envelope family.

Intent:

- preserve board-to-board internals from canonical `ICB/HB` geometry
- replace only the external/coastal envelope
- rebuild masks/outlines/edges/facilities from that family

Priority focus:

- England ICBs are the main target because they currently show the strongest inland-water projection problem
- Wales is secondary
- Scotland and Northern Ireland can remain closer to the current source path initially if needed

### 2. `bsc` external-envelope product

Use the more strongly simplified official `BSC` envelope family for outer coast comparison.

Intent:

- compare the coarse official coastal character directly
- see whether that simpler family is visually stronger than the `bfe` envelope approach
- measure what is lost at external coastal fidelity

## Current Source Hypothesis

### England

- `bfe`:
  - `NHS England Regions (January 2024) EN BFE`

  - `NHS England Regions (January 2024) EN BSC`

### Wales

Wales now uses:

- `bfe`:
  - `Local Health Boards (December 2023) Boundaries WA BFE`
- `bsc`:
  - `Countries (December 2023) UK BSC` fallback envelope for Wales

### Scotland / Northern Ireland

These continue to keep canonical internals, but use official UK country envelopes externally:

- `bfe`:
  - `Countries (December 2023) UK BFE`
- `bsc`:
  - `Countries (December 2023) UK BSC`

## What To Build

Both compare products should emit:

1. board polygons
2. topology/shared-edge products
3. dissolved Region outline products
4. land/sea mask products
5. facility containment/snap validation

Important implementation rule:

- compare families are built side-by-side under `public/data/compare/<productId>/`
- runtime swapping should only change which prepared map-product files are read
- compare swapping must not rewrite the live baseline family or mutate the facilities file

The compare products should be materialized side by side, not cut over immediately.
